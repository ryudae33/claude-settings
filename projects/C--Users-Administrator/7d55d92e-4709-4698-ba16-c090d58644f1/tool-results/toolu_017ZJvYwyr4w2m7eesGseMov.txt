  1165→2.4 Operator Symbols
  1166→
  1167→ The operator table is listed below.
  1168→ The calculation follows the precedence of operator first then the associativity.
  1169→
  1170→    For example
  1171→
  1172→             left-to-right associativity
  1173→
  1174→                    A=A*B/C%D                                                (A=(((A*B)/C)%D))
  1175→                                   1
  1176→                                      2
  1177→                                          3
  1178→                                       4
  1179→
  1180→             right-to-left associativity                                     ( A -= ( B += ( 10 && ( ! (!D) ) ) ) )
  1181→
  1182→                   A -= B += 10 && !1 !D
  1183→                                                    2
  1184→
  1185→                                                 3
  1186→                                            4
  1187→                                       5
  1188→
  1189→ The calculation will proceed by the type of the operand.
  1190→
  1191→    1. When both values come as the integer type, the calculation will proceed by the integer
  1192→
  1193→    type such as
  1194→
  1195→             int var_a = 10
  1196→
  1197→             int var_b = 3
  1198→
  1199→             float var_c = var_a / var_b
  1200→
  1201→                 By the operator priority, the calculation goes / first and then =.
  1202→
  1203→                 var_a / var_b = 10 / 3 = 3  (both var_a and var_b are integers)

  1476→2.6 Endianness and Conversion
  1477→
  1478→      Endianness refers to the applications in the memory or the communication networks in which
  1479→data comes in multiple bytes for the expressions of the order to sort among multiple bytes because
  1480→the minimal unit comes in bytes.
  1481→
  1482→      Little Endian. Place the low bits of the multibyte at smaller addresses and the high bits at
  1483→           larger ones.
  1484→                  32-bit integer
  1485→
  1486→                   Big Endian. Place the high bits of the multibyte at smaller addresses and the
  1487→               low bits at larger ones.
  1488→
  1489→                                    32-bit integer
  1490→
  1491→      It is a factor to consider because different orders to sort may cause different conversion results.
  1492→For example, an integer type is 32 bits. Namely, it occupies 4 bytes. If the int value is 300, the
  1493→expression in 16-bit is 0x0000012C. If using Little Endian for memory storing or communication
  1494→networks, the order is
  1495→
  1496→                  [0] [1] [2] [3]
  1497→                  2C 01 00 00
  1498→      It fetches 0x0000012C if adopting Little Endian while 0x2C010000 if Big Endian. These are two
  1499→completely different values, which may cause different results.
  1500→
  1501→      TMscript provides a variety of robot programming functions. Among the functions, the
  1502→conversions of the numeric types, such as int, float, and double, also follow the general
  1503→endianness rules, which usually apply to file functions, communication-related functions (such as
  1504→SerialPort, Socket, Modbus), or value-to-byte conversion-related functions. Unless specified
  1505→otherwise, the function defaults to go by Little Endian. For the string types, it goes by the UTF-8
  1506→encoding format that features ASCII character compatibility and fixes endianness without the byte
  1507→order distinction. For the bool types, it turns the Boolean value true to 1 and false to 0. Refer to

  1863→3.5 Customized Function
  1864→
  1865→      Other than the built-in define, main, closetop, and errorstop functions, Script projects provide
  1866→customized functions for users. As customized to the Show() function below, there is no parameter
  1867→to input nor the value to return (void).
  1868→
  1869→      For example:
  1870→            main
  1871→            {
  1872→                  Show() // Call the customized function Show().
  1873→            }
  1874→            void Show()
  1875→            {
  1876→                  Display("Hello TM AI Cobot")
  1877→            }
  1878→
  1879→      Customized functions support parameter input and output. There are two functions
  1880→customized in the example below. The Add() can add up the two values users input and return the
  1881→added up result; the Sub() subtracts the two values users input and return the subtraction result.
  1882→Note that as long as the returned value is not void, it requires to return the value of the same type
  1883→as the definition. Such as the Add() adopts the float type, and the returned variable must be the
  1884→float type as well.
  1885→
  1886→      For example:
  1887→            main
  1888→            {
  1889→                  string Add_Result = "Add:" + Add(5.9, 3.6)
  1890→                  string Sub_Result = "Sub:" + Sub(5.9, 3.6)
  1891→                  Display("Green", "Yellow", "Result", Add_Result + newline + Sub_Result)
  1892→            }
  1893→            float Add(float augend, float addend)
  1894→            {
  1895→                  float result = augend + addend
  1896→                  return result
  1897→            }
  1898→            float Sub(float minuend, float subtrahend)
  1899→            {
  1900→                  float result = minuend - subtrahend
  1901→                  return result
  1902→            }
  1903→
  1904→                         Important
  1905→
  1906→                              If the function defines the return type, it must return with the same type. If the definition is
  1907→                              void, it is not required to use return.

  1916→      Use part of the code as a comment. The compiler ignores the commented code. Users can

  1926→

  1937→            {
  1938→
  1939→                  /*

  1950→3.7 Variable
  1951→
  1952→      By variable declaration, users can proceed with applications such as calculation, reading and
  1953→writing, and parameter conveyance. Variables come with the global variables and the local
  1954→variables.
  1955→
  1956→                    Important
  1957→
  1958→                    Variables are case sensitive such as TMrobot and TMROBOT are two different variables.
  1959→
  1960→      Project Variable
  1961→
  1962→      Once declared a variable in the definition section, the variable is a global variable in the
  1963→project. This variable can be read and written in functions of the same project. As shown below, if
  1964→the define declares the variable b, and the main function runs, it calls Test1() first and sets the
  1965→variable as 20. When it displays the variable value next, it gets b as 20 for the last set value.
  1966→
  1967→            define
  1968→            {
  1969→
  1970→                  int b = 0
  1971→            }
  1972→            main
  1973→            {
  1974→
  1975→                  Test1()
  1976→                  Display("B Value: " + b)
  1977→            }
  1978→            void Test1()
  1979→            {
  1980→                  b = 20
  1981→            }
  1982→            // B Value: 20
  1983→
  1984→                         Important
  1985→
  1986→                              The project variables in the project are different from the global variables in the global
  1987→                              variable manager of the robot. The global variables in the project indicate they can operate
  1988→                              across functions in the project. Other than across functions in the project, the global
  1989→                              variables in the global variable manager of the robot operate across functions of different
  1990→                              projects.
  1991→
  1992→                         Important
  1993→
  1994→                              Users can create global variables in the global variable manager in the TMflow operation
  1995→                              interface only but not with TMscript syntax.
  1996→
  1997→      Local Variable
  1998→
  1999→      If variables are declared outside of the definition section such as main, closestop, errorstop,
  2000→and customized functions, the variables are local variables and good in the functions only.
