-## 2. 시스템 구조 이해
-
-### 2.1 데이터 흐름
-
-```
-[품번 스캔]
-    ↓
-[Table_Part에서 WorkID 조회]
-    ↓
-[Table_Seq에서 해당 WorkID의 시퀀스 로드]
-    ↓
-[시퀀스 순서대로 작업 진행]
-```
-
-### 2.2 핵심 개념
-
-#### WorkID란?
-- **품번(PartNo)**과 **작업 시퀀스(Seq)**를 연결하는 키
-- Table_Part의 `WorkID_OP10`, `WorkID_OP20` 등의 컬럼에 저장
-- 예: 품번 `88571-DC010`의 OP10 공정 WorkID가 `1`이면, Table_Seq에서 WorkID=1인 시퀀스 로드
-
-#### 공정별 컬럼명 규칙
-- OP10 공정: `WorkID_OP10`, `WorkStandard_OP10`
-- OP20 공정: `WorkID_OP20`, `WorkStandard_OP20`
-- ... (공정별로 컬럼이 분리됨)

-| **DELETE** | 선택한 품번 삭제 |

-## 6. 실전 예제
-
-### 6.1 새 품번 추가 (전체 과정)
-
-**상황:** 신규 품번 `88571-DC100` 추가
-
-#### Step 1: Part 추가
-1. TablePartEditor 또는 PopV4 FrmPart 열기
-2. 새 행 추가
-3. 입력:
-   - PartNo: `88571-DC100`
-   - PartName: `SEAT ASSY-RR,LH SWING NEW`
-   - Pos1: `FR`
-   - Pos2: `LH`
-   - type: `SWING`
-   - WorkID_OP10: `1` (기존 시퀀스 사용)
-4. Target 바코드 설정:
-   - TargetMotorBarcode: `98350-DC100`
-   - TargetFrameBarcode: `88571-DC100`
-5. 저장
-
-#### Step 2: 시퀀스 확인
-- WorkID `1`의 시퀀스가 적합한지 FrmSeq에서 확인
-- 필요시 새 WorkID 생성
-
-### 6.2 새 시퀀스 생성
-
-**상황:** 스캔 항목이 다른 새 작업 시퀀스 필요
-
-#### Step 1: 새 WorkID 생성
-1. FrmSeq 열기
-2. **[New ID]** 클릭 → 새 WorkID 자동 생성 (예: `5`)
-
-#### Step 2: 시퀀스 구성
-1. 첫 번째 행에 첫 작업 입력:
-   - MESSAGE: `모터 바코드 스캔`
-   - Type: `SCAN`
-   - Target: `SVR/TargetMotorBarcode`
-2. **[Insert]**로 다음 시퀀스 추가
-3. 모든 시퀀스 입력 후 **[Save]**
-
-#### Step 3: Part에 연결
-1. TablePartEditor 열기
-2. 해당 품번의 WorkID_OP10을 `5`로 변경
-3. 저장
-
-### 6.3 기존 시퀀스 복사
-
-**상황:** WorkID `1`을 복사하여 `6`으로 만들기
-
-1. FrmSeq 열기
-2. WorkID 콤보박스에서 `1` 선택 → **[Load]**
-3. 복사대상 콤보박스에서 `6` 입력
-4. **[Copy]** 클릭
-5. WorkID `6` 로드 후 필요한 부분만 수정
-6. **[Save]**
-
-### 6.4 스캔 타겟 변경
-
-**상황:** 특정 품번의 모터 바코드 타겟 변경
-
-1. TablePartEditor 열기
-2. 해당 품번 검색
-3. `TargetMotorBarcode` 컬럼 값 수정
-4. **[저장]**

-## 7. 문제 해결
-
-### 7.1 자주 발생하는 문제
-
-#### 문제: 품번 스캔 후 "Empty Info" 메시지
-
-**원인:** Table_Part에 해당 품번이 없음
-
-**해결:**
-1. TablePartEditor에서 품번 존재 확인
-2. 없으면 새로 추가
-3. 품번 11자리 확인 (앞 11자리로 매칭됨)
-
-#### 문제: 시퀀스 로드 안됨
-
-**원인:** WorkID가 잘못 설정됨
-
-**해결:**
-1. FrmPart에서 해당 품번의 WorkID 확인
-2. FrmSeq에서 해당 WorkID 존재 확인
-3. WorkID 값 수정 후 저장
-
-#### 문제: 스캔 시 항상 NG
-
-**원인:** Target 설정 오류
-
-**해결:**
-1. FrmSeq에서 해당 시퀀스의 Target 확인
-2. `SVR/컬럼명` 형식인 경우 TablePartEditor에서 해당 컬럼값 확인
-3. 바코드 형식(앞자리) 일치 확인
-
-#### 문제: DB 컬럼 없음 오류
-
-**원인:** DataSave 체크했으나 컬럼 미생성
-
-**해결:**
-1. FrmSeq 열기
-2. 해당 WorkID 로드
-3. **[Column Check]** 버튼 클릭
-4. 컬럼 자동 생성 확인
-
-### 7.2 로그 확인
-
-문제 발생 시 메인 화면 하단 로그 창 확인:
-- `LoadPart-품번-RecordCount:0` → 품번 없음
-- `LoadWorkID-...Value:1` → WorkID 로드됨
